How it works. Each ticket consumes 64 mouse events. Those events — relative timestamps, deltas, x/y coordinates, button state — get packed into bytes, your optional note is appended, and the whole thing is hashed with SHA-512. The digest is read in 16-bit chunks, biased values are rejected, and what's left is mapped cleanly onto the game's number ranges.

What it isn't. A way to win. The numbers come out flat across their range; the lottery is still the lottery. This is a nicer ritual than a quick-pick, not a strategy.
